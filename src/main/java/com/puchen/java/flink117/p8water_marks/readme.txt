1.乱序和迟到的区别
    乱序: 数据的顺序乱了，时间小的比时间大的晚来
    迟到: 数据的时间戳 < 当前的watermark
2.乱序，迟到数据的处理
    1.watermark中指定乱序等待的时间
    2.如果开窗设置窗口允许迟到时间
    3.关窗后可以放入侧输出流中

如果watermark等待3s，窗口允许迟到2s，为什么不直接watermark等待5s 或者窗口允许迟到5s?
    1.watermark等待时间不会设置太大，影响的计算延迟
        如果3s  窗口第一次触发计算和输出 13s的数据来  13-3=10s
        如果5s  窗口第一次触发计算和输出 15s的数据来  15-5=10s

    2.窗口允许迟到 是对大部分迟到的数据处理 尽量然给结果准确，修正的结果。
        如果只设置允许迟到5s，那么就会导致频繁重新输出
总结:(设置经验)
1.water等待时间，设置一个不算特别大的，一般是秒级，在乱序和延迟取舍。
2.设置一定的窗口允许迟到，只考虑大部分的迟到的数据 极端小部分迟到很久的数据 不管
3.极端小部分迟到很久的数，放入侧输出流。 获取到之后可以做各种处理。
